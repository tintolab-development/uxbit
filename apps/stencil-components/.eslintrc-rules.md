# ESLint 규칙 (커스텀)

> **목적**: 무한 루프 및 일반적인 오류를 방지하기 위한 ESLint 규칙 제안

## 제안하는 규칙

### 1. render() 메서드에서 prop/state 변경 금지

```javascript
// .eslintrc.js에 추가할 규칙
rules: {
  'no-state-update-in-render': 'error', // 커스텀 규칙 필요
}
```

**검출 패턴**:

- `render()` 메서드 내에서 `this.prop = ...` 또는 `this.state = ...` 패턴

### 2. componentDidUpdate()에서 조건부 업데이트 필수

**수동 검토 필요** (ESLint로 자동화 어려움):

- `componentDidUpdate()`에서 변경 감지 로직 확인
- 무조건 업데이트하는 패턴 경고

---

**참고**: Stencil 특화 규칙은 커스텀 ESLint 플러그인 개발이 필요합니다.
