# 화면 와이어프레임 (Text Wireframe)

화면ID: PROMO_BANNER_003  
화면명: 프로모션 배너 상세 화면

## 1) 화면 목적

- 프로모션 배너 클릭 시 상세 정보 제공
- GA4 이벤트 기반 클릭률 측정 (15% 목표)
- 외부 링크/내부 섹션 이동 경로 제어

## 2) 레이아웃 구조 (Top → Bottom)

- **Header**:
  - 뒤로가기 버튼 (터치 영역 44x44px)
- **Body**:
  - 대형 이미지 (반응형 WebP, 최소 300x200px)
  - 설명 텍스트 (최대 2줄, 폰트 크기 16px)
  - CTA 버튼 (색상: 브랜드 주색상, 폰트 크기 18px)
- **Footer**: 없음

## 3) 컴포넌트 상세

### 1. 대형 이미지

- **조건**: `GET /api/banner/{id}` 응답 성공 시
- **기술 사양**:
  - WebP 포맷, 반응형 크기 (최소 300x200px)
  - 로딩 중 스피너 표시 (3초 이상 지연 시)
- **UX 정책**:
  - 이미지 대비 텍스트 대비도 4.5:1 이상 (WCAG 2.1 준수)

### 2. 설명 텍스트

- **조건**: 항상 표시
- **기술 사양**:
  - 최대 2줄, 자동 줄바꿈
  - 폰트: `Roboto`, 16px, `#333`
- **UX 정책**:
  - 텍스트 대비 배경 대비도 4.5:1 이상

### 3. CTA 버튼

- **조건**: 항상 표시
- **기술 사양**:
  - 터치 영역 44x44px
  - 클릭 시 `banner_click` GA4 이벤트 발생
- **동작 로직**:
  - 외부 링크: `window.open(url, '_blank')`
  - 내부 섹션: `navigate.to(sectionId)`
- **UX 정책**:
  - 클릭률 15% 목표 (A/B 테스트 대상)

## 4) 상태 (States)

| 상태        | 조건                             | 동작                           |
| ----------- | -------------------------------- | ------------------------------ |
| **Success** | `GET /api/banner/{id}` 응답 성공 | GA4 `banner_click` 이벤트 발생 |
| **Error**   | 잘못된 ID 또는 API 실패          | `ERROR_404_004` 화면 전환      |

## 5) 인터랙션 흐름

1. `HOME_001` 프로모션 배너 클릭 →
2. `GET /api/banner/{id}` 호출 →
3. 성공 시 PROMO_BANNER_003 표시, 실패 시 ERROR_404_004 표시 →
4. CTA 버튼 클릭 시 외부/내부 경로 이동

## 6) 데이터 & API

- **주요 API**:
  - `GET /api/banner/{id}` (화면 진입 시 호출)
  - `GET /api/cache/banner` (최근 3개 캐시 사용, 실패 시)
- **에러 처리**:
  - 3회 실패 시 `ERROR_404_004` 진입
  - GA4 `error_404` 이벤트 기록
- **캐시 정책**:
  - 실패 시 최근 3개 캐시 사용 (최대 24시간 유효)

## 7) 검증 항목

1. CTA 버튼 터치 영역 44x44px 이상
2. WebP 이미지 로딩 시 3초 이상 지연 시 스피너 표시
3. GA4 이벤트 `banner_click` 정상 기록
4. 잘못된 ID 시 ERROR_404_004 정상 전환
