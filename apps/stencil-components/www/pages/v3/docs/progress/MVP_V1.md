## 🎯 MVP 개요

### MVP 목표

싱글페이지 이커머스 랜딩페이지의 핵심 기능을 최소한으로 구현하여 사용자가 상품을 탐색하고 검색할 수 있는 기본 경험을 제공합니다.

### MVP 핵심 기능

1. **홈 화면 (HOME_001)** ✅
   - 프로모션 배너 슬라이드 (자동 재생 5초 간격)
   - 검색바 (화면 상단 15% 내 위치)
   - 하단 내비게이션 바 (홈/검색/프로필/설정)

2. **검색 화면 (SEARCH_002)** ✅
   - 실시간 검색 (debounce 300ms)
   - 검색 결과 목록 (최대 10개)
   - 검색 결과 클릭 시 상품 상세 모달 표시

3. **프로모션 배너 상세 (PROMO_BANNER_003)** ✅
   - 배너 상세 이미지
   - 설명 텍스트
   - CTA 버튼

4. **오류 처리 (ERROR_404_004)** ✅
   - 404 오류 UI
   - 홈으로 돌아가기 버튼

### MVP 성공 지표

- 검색바 가시성: 화면 상단 15% 내 위치, 5초 이내 인식률 90%
- 프로모션 클릭률: 15% 이상
- 터치 타겟 크기: 모든 클릭 영역 44x44px 이상
- WCAG 2.1 AA 준수

## 📱 화면 구성

### 싱글페이지 구조

```
┌─────────────────────────────────────┐
│  tinto-app-route (Safe area)        │
│  ┌───────────────────────────────┐  │
│  │  검색바 (상단 15%)            │  │
│  ├───────────────────────────────┤  │
│  │                               │  │
│  │  프로모션 배너 (tinto-carousel)│  │
│  │                               │  │
│  │  검색 결과 / 배너 상세         │  │
│  │  (동적 컨텐츠 영역)            │  │
│  │                               │  │
│  └───────────────────────────────┘  │
│  tinto-navigation (하단 고정)       │
└─────────────────────────────────────┘
```

### 화면 전환 흐름

```
HOME_001 (홈)
  ├─ 검색바 클릭 → SEARCH_002 (검색)
  ├─ 배너 클릭 → PROMO_BANNER_003 (배너 상세)
  └─ 내비게이션 클릭 → 해당 섹션 이동

SEARCH_002 (검색)
  ├─ 검색 결과 클릭 → 상품 상세 모달 (tinto-modal)
  └─ 뒤로가기 → HOME_001

PROMO_BANNER_003 (배너 상세)
  ├─ CTA 버튼 클릭 → 외부 링크 또는 내부 섹션
  └─ 뒤로가기 → HOME_001

ERROR_404_004 (오류)
  └─ 홈으로 가기 → HOME_001
```

## 🖼️ 화면별 상세 스펙

### 1. 홈 화면 (HOME_001) ✅

#### 🎨 디자이너 관점

**레이아웃 구조**:

- **Header 영역** (화면 상단 15%)
  - `tinto-search-bar`: 전체 너비, 상단 고정
  - 배경색: `#FFFFFF`
  - 그림자: `0 2px 8px rgba(0,0,0,0.1)`

- **Body 영역**
  - `tinto-carousel`: 프로모션 배너 슬라이드
    - 자동 재생: 5초 간격
    - 인디케이터: 하단 중앙
    - 이미지 비율: 16:9
    - 높이: 화면 너비의 50%
  - `tinto-section`: 컨텐츠 영역
    - 패딩: `16px`
    - 배경색: `#F5F5F5`

- **Footer 영역** (하단 고정)
  - `tinto-navigation`: 4개 아이콘
    - 아이콘 간격: `24px`
    - 활성 상태: 브랜드 주색상
    - 비활성 상태: `#999999`

#### 📊 PM 관점

**GA4 이벤트**:

- `home_view`: 화면 진입 시
- `banner_click`: 배너 클릭 시 (배너 ID 포함)
- `search_bar_click`: 검색바 클릭 시
- `navigation_click`: 내비게이션 아이콘 클릭 시

**KPI 측정**:

- 검색바 클릭률: 검색바 클릭 수 / 홈 화면 조회 수
- 배너 클릭률: 배너 클릭 수 / 배너 노출 수 (목표: 15%)
- 내비게이션 사용률: 각 아이콘 클릭 수

#### 🛠️ 기획자 관점

**기능 요구사항**:

1. 화면 진입 시 mock 데이터로 프로모션 배너 3개 표시
2. 검색바 클릭 시 검색 화면으로 전환
3. 배너 클릭 시 배너 상세 화면 표시
4. 하단 내비게이션으로 화면 간 이동

**사용자 플로우**:

```
사용자 진입
  → 홈 화면 로드 (mock 배너 데이터)
  → 검색바 클릭 → 검색 화면
  → 배너 클릭 → 배너 상세
  → 내비게이션 클릭 → 해당 섹션
```

**상태 관리**:

- 로딩 상태: `tinto-loading` 표시 (초기 로드 시)
- 에러 상태: `tinto-toast` 에러 메시지 표시
- 빈 상태: 배너 없을 시 빈 화면 표시

### 2. 검색 화면 (SEARCH_002) ✅

#### 🎨 디자이너 관점

**레이아웃 구조**:

- **Header 영역**
  - 검색 입력 필드 (전체 너비)
  - 자동 포커스

- **Body 영역**
  - 검색 결과 리스트
    - 최대 10개 항목
    - 각 항목: `tinto-card` 사용
    - 항목 간 간격: `12px`
    - 패딩: `16px`

#### 📊 PM 관점

**GA4 이벤트**:

- `search_input`: 검색어 입력 시 (디바운스 300ms 후)
- `search_submit`: 검색 제출 시
- `search_result_click`: 검색 결과 클릭 시 (결과 ID 포함)
- `search_no_result`: 검색 결과 없음 시

**KPI 측정**:

- 검색 전환률: 검색 제출 수 / 검색바 클릭 수
- 검색 결과 클릭률: 결과 클릭 수 / 검색 제출 수
- 평균 검색어 길이

#### 🛠️ 기획자 관점

**기능 요구사항**:

1. 검색어 입력 시 300ms 디바운스 후 mock 데이터 검색
2. 검색 결과 최대 10개 표시
3. 검색 결과 클릭 시 상품 상세 모달 표시
4. 검색 결과 없을 시 빈 상태 메시지 표시

**사용자 플로우**:

```
검색바 클릭
  → 검색 화면 진입 (자동 포커스)
  → 검색어 입력 (300ms 디바운스)
  → 검색 결과 표시 (mock 데이터)
  → 결과 클릭 → 상품 상세 모달
```

## 📝 테스트 가이드

### 로컬 테스트 방법

1. **개발 서버 실행**

   ```bash
   npm run start
   # 또는
   pnpm start
   ```

2. **페이지 접속**
   - URL: `http://localhost:3334/pages/v3/`

3. **기능 테스트**
   - 홈 화면에서 배너 확인
   - 검색바 클릭 → 검색 화면 전환
   - 검색어 입력 (노트북, 스마트폰, 태블릿)
   - 검색 결과 클릭 → 상품 상세 모달
   - 배너 클릭 → 배너 상세 화면
   - 내비게이션 클릭 → 화면 전환

4. **콘솔 확인**
   - 브라우저 개발자 도구 콘솔에서 GA4 이벤트 로그 확인
   - 오류 메시지 확인

### Mock 데이터 테스트

**검색 가능한 키워드**:

- `노트북` - 2개 결과
- `스마트폰` - 1개 결과
- `태블릿` - 1개 결과
- 기타 키워드 - 결과 없음

**배너 ID**:

- `banner-1` - 신상품 출시
- `banner-2` - 특가 할인
- `banner-3` - 이벤트 진행중

**상품 ID**:

- `product-1` - 맥북 프로 16인치
- `product-2` - LG 그램 17인치
- `product-3` - 아이폰 15 프로
- `product-4` - 아이패드 프로 12.9인치
