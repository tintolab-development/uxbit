# MCP 서버 테스트 체크리스트

## Phase 1: 현재 상태 테스트

### 사전 준비

- [ ] 빌드 완료 확인

  ```bash
  pnpm mcp:build
  ```

- [ ] Cursor 설정 확인
  - 설정 파일 위치 확인
  - 경로가 올바른지 확인
  - 서버 등록 확인

- [ ] Cursor 재시작
  - 완전히 종료 후 재시작
  - MCP 서버 연결 확인

### 기본 기능 테스트

#### 1. 컴포넌트 목록 조회

- [ ] 요청: "UXBIT 컴포넌트 목록을 보여줘"
- [ ] 결과: JSON 형식으로 컴포넌트 목록 반환
- [ ] 확인: 모든 컴포넌트가 포함되어 있는지

#### 2. 컴포넌트 검색

- [ ] 요청: "버튼 컴포넌트를 검색해줘"
- [ ] 결과: `tinto-button` 컴포넌트가 검색 결과에 포함
- [ ] 확인: 검색 결과가 정확한지

#### 3. 컴포넌트 상세 정보

- [ ] 요청: "tinto-button 컴포넌트의 상세 정보를 알려줘"
- [ ] 결과: Props, Events, Methods, 문서 정보 반환
- [ ] 확인: 모든 정보가 정확한지

#### 4. Props 검색

- [ ] 요청: "variant prop을 가진 컴포넌트를 찾아줘"
- [ ] 결과: variant prop을 가진 컴포넌트 목록 반환
- [ ] 확인: 결과가 정확한지

#### 5. Events 검색

- [ ] 요청: "click 이벤트를 발생시키는 컴포넌트를 찾아줘"
- [ ] 결과: click 관련 이벤트를 가진 컴포넌트 목록 반환
- [ ] 확인: 결과가 정확한지

#### 6. Methods 검색

- [ ] 요청: "show 메서드를 가진 컴포넌트를 찾아줘"
- [ ] 결과: show 메서드를 가진 컴포넌트 목록 반환
- [ ] 확인: 결과가 정확한지

#### 7. 통계 정보

- [ ] 요청: "UXBIT 컴포넌트 라이브러리 통계를 보여줘"
- [ ] 결과: 통계 정보 반환
- [ ] 확인: 통계가 정확한지

#### 8. 헬스 체크

- [ ] 요청: "서버 상태를 확인해줘"
- [ ] 결과: 헬스 체크 결과 반환
- [ ] 확인: 상태가 "healthy"인지

### 에러 처리 테스트

- [ ] 존재하지 않는 컴포넌트 조회
  - 요청: "존재하지않는-컴포넌트의 정보를 알려줘"
  - 결과: 명확한 에러 메시지 반환

- [ ] 잘못된 파라미터
  - 요청: "컴포넌트를 검색해줘" (query 없음)
  - 결과: 파라미터 필요 에러 메시지

### 성능 테스트

- [ ] 첫 요청 응답 시간
  - 첫 요청: 캐시 미스 예상
  - 응답 시간: 1초 이내 권장

- [ ] 두 번째 요청 응답 시간
  - 두 번째 요청: 캐시 히트 예상
  - 응답 시간: 100ms 이내 권장

- [ ] 동시 요청 처리
  - 여러 요청 동시 실행
  - 모든 요청이 정상 처리되는지

### 파일 감시 테스트

- [ ] custom-elements.json 변경 감지
  - 파일 수정 후 다음 요청 시 최신 데이터 반영 확인

- [ ] 문서 파일 변경 감지
  - 문서 수정 후 다음 요청 시 최신 문서 반영 확인
