# UXBIT MCP Server Vision

## 목적

UXBIT MCP Server는 **Ant Design, Material-UI, Chakra UI**와 같은 범용적인 **Headless UI 디자인 시스템**을 목표로 하는 UXBIT 컴포넌트 라이브러리를 AI 모델이 이해하고 활용할 수 있도록 지원합니다.

## 비전

### 범용성 (Universal)

- **프레임워크 독립적**: React, Vue, Angular, Svelte, Vanilla JS 등 어떤 프레임워크에서도 사용 가능
- **Web Components 표준**: W3C 표준을 준수하여 브라우저 네이티브 지원
- **재사용성**: 한 번 작성한 컴포넌트를 다양한 프로젝트에서 활용

### Headless 아키텍처

- **스타일과 로직 분리**: 비즈니스 로직과 시각적 표현의 명확한 분리
- **디자인 토큰 시스템**: 커스터마이징 가능한 테마 및 디자인 변수
- **컴포지션 가능**: 작은 컴포넌트를 조합하여 복잡한 UI 구성

### 확장성

- **다양한 프로젝트 적용**: 마케팅 사이트, 제품 대시보드, 모바일 앱 등
- **디자인 시스템 통합**: 기존 디자인 시스템과의 호환성
- **점진적 도입**: 필요한 컴포넌트만 선택적으로 사용 가능

## 비교: 다른 디자인 시스템과의 차이점

| 특징         | Ant Design | Material-UI | Chakra UI  | **UXBIT**              |
| ------------ | ---------- | ----------- | ---------- | ---------------------- |
| 프레임워크   | React      | React       | React      | **Web Components**     |
| 범용성       | React 전용 | React 전용  | React 전용 | **프레임워크 독립**    |
| 스타일링     | CSS-in-JS  | CSS-in-JS   | CSS-in-JS  | **CSS Variables**      |
| 번들 크기    | 큰 편      | 큰 편       | 중간       | **작음 (필요한 것만)** |
| 커스터마이징 | 제한적     | 제한적      | 유연함     | **매우 유연**          |

## MCP 서버의 역할

이 MCP 서버는 AI 모델이 다음과 같은 작업을 수행할 수 있도록 지원합니다:

1. **컴포넌트 검색**: 프로젝트에 적합한 컴포넌트 빠르게 찾기
2. **API 이해**: Props, Events, Methods를 통한 정확한 사용법 파악
3. **일관성 유지**: 디자인 시스템 가이드라인에 따른 일관된 UI 구성
4. **문서 참조**: 상세한 사용 예제와 가이드 제공

## 사용 사례

### 1. AI 기반 컴포넌트 개발

```
사용자: "버튼 컴포넌트를 만들어줘"
AI: UXBIT의 tinto-button 컴포넌트 정보를 조회하고 적절한 사용법 제안
```

### 2. 디자인 시스템 일관성 유지

```
사용자: "프로젝트에 일관된 카드 컴포넌트를 사용하고 싶어"
AI: UXBIT의 tinto-card 컴포넌트와 관련 컴포넌트들을 검색하여 제안
```

### 3. 프레임워크 마이그레이션

```
사용자: "React에서 Vue로 마이그레이션할 때 컴포넌트는?"
AI: UXBIT은 Web Components이므로 프레임워크 변경 없이 동일하게 사용 가능
```

## 목표

- ✅ **범용성**: 모든 프레임워크에서 사용 가능한 컴포넌트 라이브러리
- ✅ **Headless**: 스타일과 로직의 명확한 분리
- ✅ **확장성**: 다양한 프로젝트에 적용 가능
- ✅ **AI 친화적**: MCP를 통한 AI 모델과의 원활한 협업
